# Монтирование VK Cloud (S3) как сетевого диска в Windows через rclone

Простая и удобная инструкция по подключению хранилища VK Cloud как обычного диска (например, `V:`) в Windows с помощью **rclone** + **WinFsp**.

Способ идельно подходит для врмеенного хранения любых файлов, либо быстрой передачи файлов в пару кликов. Например я вставил фото и видео ниже именно с моего веб-хранилеща 

---

## 🎯 Что даёт этот способ

- Хранилище VK Cloud работает как обычный диск в Проводнике
- Кнопки в меню навигации ПКМ для копирования прямой web-ссылки на файл 
- Поддержка кэширования, быстрого чтения и записи
- Автоматическое монтирование при старте Windows (можно настроить)
- Полная совместимость с S3 API VK Cloud

## 💩 Я выбрал ВК потому что (удивительно но у них):
- Удобная привязка карты и автоплатежи (для РФ актуально)
- Оплата по факту использования (1Гб постоянного хранения = ~1 руб/мес)
- Простая и быстрая регистрация, удобный интерфейс, бонусные рубли при регистрации

# 🚀 Установка

## Запуск скрипта
*Перед запуском скрипта необходимо выполнить инструкции из блока [Сначала зависимости](#сначала-зависимости)*

1. Скачайте файл VKDiskMenu.ps1
2. Поместите в папку `C:\rclone`
3. Откройте файл и отредактируйте настройки в самом начале

```ps1
$DOMAIN        = "storage.cabi.world"    # под ваши настройки VK Object Storage
$BUCKET        = "vk-disk"               # под ваши настройки VK Object Storage
$DRIVE_LETTER  = "V:"                    # любой
$RCLONE_REMOTE = "VKDisk"                # под ваши настройки rclone
$ENDPOINT_HOST = "hb.bizmrg.com"         # без изменений
$REGION        = "ru-msk"                # без изменений
```

**ВАЖНО: У ВАС ДОЛЖЕН БЫТЬ СВОЙ ДОМЕН**

*Скрипт генерирует ссылку приметивно: подставлет ваше значение из $DOMAIN к https конструкции.*

*Следовательно: Вам необходимо в настройках бакета VK Object Storage добавить свой домен и попросить тех.поддержку сделать SSL сертификат для него. У них на сайте написано, как всё это сделать.*

5. Выполните команду ниже (начнется выполнение скриптов, займет ~2-3 минуты)

```bash
powershell -ExecutionPolicy Bypass -File "C:\rclone\VKDiskMenu.ps1" install
```

## Сначала зависимости 

### 1. Установка WinFsp
1. Скачайте последнюю версию с официального сайта: [https://winfsp.dev/rel/](https://winfsp.dev/rel/)
2. Установите как обычную программу.

### 2. Установка rclone

1. Скачайте **rclone** для Windows: [https://rclone.org/downloads/](https://rclone.org/downloads/)
2. Распакуйте архив.
3. Переместите файл `rclone.exe` в папку `C:\rclone`.

### 3. Добавление rclone в PATH

1. Нажмите `Win` → введите «**Переменные среды**» → выберите **«Изменить переменные среды системы»**.
2. Нажмите кнопку **«Переменные среды...»**.
3. В разделе **«Переменные пользователя»** найдите `Path` → нажмите **«Изменить»**.
4. Нажмите **«Создать»** и добавьте путь:  
   `C:\rclone`
5. Нажмите ОК во всех окнах.

Проверьте установку, открыв новую командную строку:

```bash
rclone version
```

## 🔧 Настройка подключения к VK Cloud

### Вариант 1: Через веб-интерфейс (рекомендуется)

```bash
rclone rcd --rc-web-gui
```

Откроется браузер. Перейдите во вкладку **Configs → Create new**.

- **Name**: `VKDisk`
- **Type**: `Amazon S3`
- **Provider**: `Any other S3 compatible provider`
- **Access Key ID**: ваш Access Key из панели VK Cloud
- **Secret Access Key**: ваш Secret Key из панели VK Cloud
- **Endpoint**: `hb.bizmrg.com`

Остальные параметры можно оставить по умолчанию.

### Вариант 2: Через консоль

```bash
rclone config
```

Следуйте инструкции:

- `n` → new remote
- `name`: `VKDisk`
- `Storage`: `4` (или `s3`)
- `provider`: `Other` (Any other S3 compatible provider)
- `access_key_id`: ваш Access Key
- `secret_access_key`: ваш Secret Key
- `endpoint`: `hb.bizmrg.com`
- Остальное — Enter (по умолчанию)

## ▶️ Монтирование диска

Основная команда для монтирования:

```bash
rclone mount VKDisk:vk-disk V: ^
  --links ^
  --vfs-cache-mode full ^
  --vfs-cache-max-size 20G ^
  --vfs-read-chunk-size 64M ^
  --vfs-read-chunk-size-limit 1G ^
  --buffer-size 128M ^
  --vfs-cache-max-age 24h ^
  --transfers 16 ^
  --vfs-cache-poll-interval 15s ^
  --dir-cache-time 10s ^
  --volname "VK Диск" ^
  --vfs-disk-space-total-size 100G ^
  --no-console
```

**Важно:**
- Замените `VKDisk:vk-disk` на ваше имя remote и название bucket'а
- Можно изменить букву диска (`V:` на любой свободный)

---

## Полезные команды

- **Проверить подключения**: `rclone listremotes`
- **Посмотреть содержимое**: `rclone ls VKDisk:vk-disk`
- **Размонтировать диск**: `rclone unmount V:`
- **Запуск с отображением лога** (для отладки): уберите `--no-console`

## Автозагрузка (по желанию)

Можно создать `.bat` файл и добавить его в автозагрузку Windows.

Создайте файл `mount_vkcloud.bat`:

```bat
@echo off
rclone mount VKDisk:vk-disk V: --links --vfs-cache-mode full --vfs-cache-max-size 20G --vfs-read-chunk-size 64M --buffer-size 128M --vfs-cache-max-age 24h --transfers 16 --no-console --vfs-cache-poll-interval 15s --dir-cache-time 10s --volname "VK Диск"
```

# Сравнение облачных хранилищ: Подписки vs S3 (VK Object Storage)

Краткий гид по выбору наиболее экономичного способа хранения данных в зависимости от их объема и частоты использования.

## 🚀 Главные выводы (TL;DR)

*   **S3 выгоднее до 80-90 ГБ**: Если ваши данные занимают малый объем, модель оплаты за каждый гигабайт (1.09 руб/мес) дешевле минимальных платных тарифов.
*   **Подписки выгоднее от 100 ГБ**: При достижении 1 ТБ обычные облака (Яндекс, Сбер) становятся в **3.5 раза дешевле** S3.
*   **S3 только для «холодных» данных**: Платный исходящий трафик (1.67 руб/ГБ) делает S3 невыгодным для активного просмотра видео или обмена файлами.
*   **Бесплатный максимум**: Если платить не хочется совсем — используйте **TeraBox (1 ТБ)** или **Mega (20 ГБ)**.

## 🔍 Подробный разбор моделей

### 1. Модель S3 (1.09 руб/ГБ + 1.67 руб трафик)
**Плюсы:**
*   Платите только за фактически занятое место (побайтовая тарификация).
*   Идеально для бэкапов, к которым редко обращаются.
*   Нет переплаты за «пустые» гигабайты в рамках тарифа.

**Минусы:**
*   Дорого при больших объемах (свыше 100 ГБ).
*   Счет растет при каждом скачивании файла.

### 2. Модель подписки (Fixed Price)
**Плюсы:**
*   Неограниченный и бесплатный трафик на скачивание.
*   Наличие дополнительных сервисов (безлимит для фото, почта без рекламы).
*   Резкое снижение цены за 1 ГБ при переходе на терабайтные тарифы.

**Минусы:**
*   Фиксированный шаг (нельзя купить 150 ГБ, только 100 или 200).
*   Риск переплаты, если используется лишь 10% от купленного объема.

---

## 💡 Рекомендация по выбору

1.  **Для личного архива фото/видео (100 ГБ+):** Выбирайте **Яндекс Диск** или **iCloud**. Бесплатный трафик и мобильные приложения делают их лидерами.
2.  **Для хранения бэкапов сайта/баз данных (до 80 ГБ):** Используйте **S3**. Это обеспечит минимальные затраты и программный доступ через API.
3.  **Для временной пересылки тяжелых файлов:** Подойдет **Telegram Premium** (файлы до 4 ГБ) или бесплатный аккаунт **Mega**.
